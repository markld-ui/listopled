# Product Vision

## Суть продукта

LISTOPLED — productized landing для авторских пледов ручной работы в форме листьев. MVP не является маркетплейсом и не является полноценным интернет-магазином.

Главный сценарий:

```text
посмотрел → рассчитал → написал мастеру
```

Сайт должен помогать маленькому handmade-бренду продавать через красоту, доверие, понятный предварительный расчет и простую связь с мастером.

## Бренд и контекст

- Бренд: ЛИСТОПЛЕД.
- Мастер: Анна Слинькова.
- Продукт: индивидуальные пледы ручной работы в форме листьев.
- Текущий сайт: `https://listopled.tilda.ws/`.

Проект должен ощущаться как реальный коммерческий инструмент, а не учебный демо-сайт.

## Цели MVP

- Усилить личный бренд мастера.
- Красиво показать работы.
- Объяснить ценность ручной работы.
- Дать клиенту предварительную цену.
- Перевести клиента в личный диалог через WhatsApp, Telegram или VK.
- Снизить число однотипных вопросов.
- Дать мастеру понятную админку.
- Дать внутреннюю аналитику без платных сервисов.

## Фаза 2

Фаза 2 — `Public Landing + Calculator Conversion Slice`.

Цель:

```text
Пользователь открывает сайт → видит красивый лендинг → понимает продукт → смотрит работы → считает примерную цену → получает CTA → может написать мастеру.
```

Калькулятор в этой фазе является частью productized landing, а не самостоятельным техническим модулем. Первый публичный срез должен связать визуальное впечатление, доверие, галерею работ, предварительный расчет и переход в личный диалог с мастером.

## Входит в MVP

- Публичный лендинг.
- Галерея работ.
- Калькулятор стоимости.
- Форма обращения.
- Переходы в WhatsApp / Telegram / VK.
- Генерация текста для WhatsApp после расчета.
- Кастомная админка.
- Управление обращениями, калькулятором, галереей, отзывами, FAQ и контактами.
- Внутренняя аналитика и графики в админке.
- Dev и production среды.
- Reverse proxy.
- Скрытый от прямого внешнего доступа backend API.
- OpenAPI в development.
- Docker Compose.
- Unit tests для бизнес-логики.

## Не входит в MVP

- Онлайн-оплата.
- Корзина.
- Личный кабинет клиента.
- Регистрация клиентов.
- Wishlist.
- История заказов клиента.
- Полноценная CRM.
- Микросервисы.
- Kafka / RabbitMQ.
- Сложные промокоды.
- WhatsApp Business API.
- Мобильное приложение.

## Post-MVP

Архитектура должна оставить место под оплату, личный кабинет, статусы заказа, уведомления, CRM, блог, SEO-страницы, промокоды и интеграции. В MVP разрешен только интерфейс `IPaymentProvider` с заглушкой, без реальной оплаты.
