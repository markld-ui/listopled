# Security And Privacy

## Главный принцип

```text
Вся бизнес-логика, которую можно подделать на frontend, должна выполняться на backend.
```

## Запрещено делать на frontend

- Финально считать цену.
- Доверять рассчитанной цене от клиента.
- Менять статус обращения без backend-авторизации.
- Определять права пользователя.
- Хранить роль пользователя.
- Хранить access token в localStorage.
- Хранить секреты.
- Решать, какие admin данные можно показывать без backend-проверки.

## Auth

JWT используется только для админки.

- Access token короткоживущий.
- Refresh token хранится в httpOnly Secure SameSite cookie.
- Refresh token хранится в БД только в виде hash.
- Logout инвалидирует refresh token.
- Все `/admin` backend endpoints требуют роль Admin.

## Passwords

- BCrypt или Argon2.
- Пароли никогда не хранить в открытом виде.
- Dev admin создается из env.

## Rate limiting

```text
POST public/inquiries: 5 запросов / 10 минут / IP
POST public/calculator/quote: 30 запросов / минуту / IP
POST admin/auth/login: 5 попыток / 15 минут / IP
POST public/analytics/events: ограничить батчами и частотой
```

## Input validation

- FluentValidation на каждый Command/Query.
- Ограничения длины строк.
- Проверка enum значений.
- Проверка активности выбранных справочников.
- Санитизация текста.
- Запрет raw HTML от пользователей.

## HTTP headers

Добавить:

- `Content-Security-Policy`;
- `X-Content-Type-Options: nosniff`;
- `X-Frame-Options: DENY`;
- `Referrer-Policy`;
- `Permissions-Policy`.

## Files

Если реализуется загрузка:

- проверять MIME;
- проверять magic bytes;
- ограничивать максимальный размер;
- принимать только JPEG/PNG/WebP;
- использовать UUID для имен файлов;
- не хранить оригинальное имя как путь;
- не отдавать исполняемые файлы.

## Brand Materials

`static/photos` используется как локальное приватное хранилище брендовых материалов проекта LISTOPLED:

- реальные фотографии;
- логотипы;
- исходники;
- будущие материалы из Figma;
- дизайн-референсы.

Содержимое `static/photos` не коммитится в git и не отправляется в Docker build context, потому что эти материалы могут быть приватными, тяжёлыми или не предназначенными для публикации. В репозитории должны оставаться только `static/photos/.gitkeep` и `static/photos/README.md`.

Если изображение нужно использовать публично, его необходимо отдельно отобрать, оптимизировать и перенести в согласованную публичную директорию frontend или загрузить в MinIO/S3 после ревью.

## Personal data

Формы собирают персональные данные, поэтому обязательно:

- чекбокс согласия;
- страница `/privacy`;
- минимизация данных;
- не логировать персональные данные без необходимости;
- не отправлять персональные данные в аналитику;
- не показывать персональные данные в ошибках.

Финальный юридический текст политики приватности должен быть согласован отдельно перед production.
